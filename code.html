Latihan Repository dan Injection

Tujuan

Pada Codelab kali ini Anda akan mempelajari bagaimana mengimplementasikan Repository dan Injection dalam aplikasi portal berita yang sudah disiapkan. Hasil latihan akan terlihat seperti ini:

Alur Latihan

Berikut alur yang akan kita lakukan dalam latihan kali ini:

  1. Membuka Starter Project
  2. Membuat Repository untuk mengatur data.
  3. Membuat Service Locator dengan nama Injection.
  4. Menyesuaikan bagian UI untuk menampilkan data.
  5. Mengimplementasikan fitur bookmark.
  6. Mengubah implementasi dengan menggunakan Coroutine (khusus Kotlin).

  2. Lihatlah struktur project starter tersebut:

    Pada dasarnya halaman Home berisi TabLayout dan ViewPager2 untuk menampilkan berita baru dan yang tersimpan. Namun, saat ini kedua data tersebut masih belum tampil. Kita akan membuatnya satu per satu pada latihan ini. Kemudian, terdapat folder remote yang siap digunakan untuk mengambil data dari API beserta dengan POJO responsnya. Selain itu, juga ada folder local yang siap digunakan untuk menyimpan data ke dalam Room beserta dengan Entity-nya. Nah, pada latihan ini kita akan membuat sebuah Repository untuk mengatur data yang disimpan, apakah berasal dari network atau dari local. Sehingga, jika digambarkan akan menjadi seperti ini:

    Pada awal latihan ini, kita menggunakan Repository untuk membuat Online-Offline App, yakni aplikasi yang bisa berjalan walaupun dalam keadaan offline. Mekanismenya yaitu ketika ada internet, data dari network akan disimpan di local. Kita akan menggunakan data local sebagai sumber utama untuk membuat Single Source of Truth (SSoT).

    Hore! Anda telah menerapkan Coroutines dalam mengambil data, baik pada Retrofit maupun pada Room. Dengan memahami ini, Anda dapat melihat kode yang dibuat menjadi lebih ringkas dan aman menggunakan Coroutines.

 

Bedah Kode

Coroutines Support pada Room dan Retrofit

Perhatikan kode pada NewsDao berikut:

  1. @Insert(onConflict = OnConflictStrategy.IGNORE)
  2. suspend fun insertNews(news: List<NewsEntity>)
  3.  
  4. @Update
  5. suspend fun updateNews(news: NewsEntity)
  6.  
  7. @Query("DELETE FROM news WHERE bookmarked = 0")
  8. suspend fun deleteAll()
  9.  
  10. @Query("SELECT EXISTS(SELECT * FROM news WHERE title = :title AND bookmarked = 1)")
  11. suspend fun isNewsBookmarked(title: String): Boolean

Pada dasarnya, Anda hanya perlu mengubah suatu fungsi menjadi suspend function untuk mendukung Coroutines. Begitu juga perhatikan kode pada ApiService berikut:

  1. @GET("top-headlines?country=id&category=science")
  2. suspend fun getNews(@Query("apiKey") apiKey: String): NewsResponse

Karena sejak versi 2.6.0 Retrofit sudah mendukung Coroutine, Anda tidak perlu menambahkan library tambahan lagi. Anda hanya perlu menghapus Call dan langsung mengembalikan kelas response seperti di atas.

Kemudian perhatikan kode pada NewsRepository berikut:

  1. val response = apiService.getNews(BuildConfig.API_KEY)
  2. val articles = response.articles
Seperti yang Anda lihat, saat memanggil service dari Retrofit, tidak diperlukan lagi callback seperti sebelumnya. Anda bisa langsung mendapatkan data response dengan lebih bersih dan ringkas.


Begitu juga pada pemanggilan Room, sekarang tidak diperlukan lagi appExecutor. Anda dapat memanggilnya langsung seperti inI:

  1. suspend fun setNewsBookmark(news: NewsEntity, bookmarkState: Boolean) {
  2.     news.isBookmarked = bookmarkState
  3.     newsDao.updateNews(news)
  4. }


LiveData Builder

Perhatikan kode berikut pada NewsRepository:

  1. fun getHeadlineNews(): LiveData<Result<List<NewsEntity>>> = liveData {
  2.    emit(Result.Loading)
  3.    try {
  4.        ...
  5.    } catch (e: Exception) {
  6.        Log.d("NewsRepository", "getHeadlineNews: ${e.message.toString()} ")
  7.        emit(Result.Error(e.message.toString()))
  8.    }
  9.    val localData: LiveData<Result<List<NewsEntity>>> = newsDao.getNews().map { Result.Success(it) }
  10.    emitSource(localData)
  11. }

Sama seperti MediatorLiveData, liveData block dapat digunakan untuk membuat LiveData baru maupun menggabungkan LiveData yang sudah ada. Apabila data sumber bukan merupakan LiveData, gunakan fungsi emit. Sedangkan apabila data sumber berupa LiveData, gunakan fungsi emitSource.

Salah satu kelebihan liveData block yaitu ia mendukung Coroutine sehingga bisa menjalankan suspend function di dalamnya. Untuk memahami lebih lanjut, Anda dapat membaca beberapa tautan berikut:
